Personas do Sistema

O sistema atende três perfis com níveis distintos de acesso e objetivos diferentes.

🏢

Administrador Master

Perfil GoodWe com acesso total. Monitora a frota completa, visualiza toda a telemetria e gerencia royalties entre estabelecimentos.

📊

Gestor do Estabelecimento

Perfil B2B. Recebe relatórios financeiros consolidados, métricas de saúde do hardware e alertas de sobrecarga em tempo real.

⚡

Cliente Final

Inicia sessões de recarga via RFID, consulta o custo estimado em tempo real, acessa histórico de sessões e acompanha a economia de CO₂ gerada.

Fundamentos

Os 4 Pilares da Solução

Cada pilar resolve um problema real da gestão de recarga em ambientes com múltiplos usuários e fontes de energia.

1
DLM

Controle de Demanda

Quando o consumo total se aproxima do limite contratado do prédio, o sistema reduz automaticamente a potência dos carregadores — cortando primeiro os usuários de menor prioridade para proteger a infraestrutura.

2
Interoperabilidade

Protocolos Abertos

O gateway traduz dados dos dispositivos físicos (MODBUS / SEMS Plus) para o formato OCPP simulado, garantindo que qualquer sistema de faturamento ou gestão possa consumir os dados sem adaptações.

3
Financeiro

Tarifação Dinâmica

O custo do kWh é calculado em tempo real considerando o horário (pico / fora de pico), o tipo de plano do usuário e a disponibilidade de energia solar no momento da sessão.

4
IA

Inteligência Aplicada

Modelo preditivo antecipa picos de consumo nas próximas horas. O Síndico Virtual converte telemetria bruta em relatórios gerenciais em linguagem natural para gestores do estabelecimento.

Algoritmo Central

Controle de Demanda (DLM) e Energia

O DLM é o coração do sistema. Ele decide em tempo real quanta potência cada veículo recebe, com base no consumo total do prédio e na prioridade de cada usuário.

📡

Coleta (pull)

Gateway consulta SEMS Plus e medidores a cada ciclo. Captura kWh, tempo e estado do conector.

🔄

Cálculo DLM

Soma demanda total das sessões ativas. Compara com o limite contratado do prédio.

⚖️

Redistribuição

Se > 95% do limite, corta potência por ordem de prioridade de plano de usuário.

💰

Tarifação

Custo calculado após sessão: kWh × tarifa(horário, plano)

Usuário Padrão (BYD Dolphin)17 kW · 80%

Rede Elétrica

# Entrada: sessões ativas, limite contratado, prioridades dos planos
    def distribuir_potencia(sessoes, limite_kw, prioridades):
        demanda_total = sum(s.requested_kw for s in sessoes)

        if demanda_total <= limite_kw:
            alocar_potencia_normal(sessoes)
        else:
            # Cortar primeiro os usuários sem prioridade
            sessoes_ord = ordenar_por_prioridade(sessoes, prioridades)
            reduzir_para_encaixar(sessoes_ord, limite_kw)

        return sessoes  # com `allocated_kw` atualizado
Auditoria

Logs OCPP Simulado

Cada evento crítico gera um registro em JSON compatível com o padrão OCPP — pronto para integrar com qualquer sistema de faturamento.

OCPP-sim · evento ao vivo JSON 
{
    "messageType": "StartTransaction",
    "timestamp":   "2026-06-15T12:34:56Z",
    "connectorId": 1,
    "transactionId": "tx-12345",
    "meterValue":  12.34,
    "userId":       "u-987",
    "notes":        "simulated"
    }
messageTypeQuando é gerado
BootNotificationAo inicializar o gateway ou reconectar um carregador
StartTransactionQuando um usuário inicia uma sessão de recarga
MeterValuesPeriodicamente durante a sessão — registra kWh acumulado
StopTransactionAo encerrar a sessão — contém resumo final de energia e custo
Guia de Uso

Passo a Passo — Como Usar

Siga esta ordem: primeiro instale, depois crie um usuário, e só então cadastre veículos e inicie sessões.

ℹ️ Pré-requisito: Python 3.8 ou superior instalado.
0

Instalar e iniciar

Clone o projeto, crie um ambiente virtual e instale as dependências.

bash 
# Entrar na pasta do projeto
    cd caminho/para/o/projeto

    # Criar ambiente virtual (recomendado)
    python -m venv .venv
    .venv\Scripts\activate      # Windows
    source .venv/bin/activate   # Linux / Mac

    # Instalar dependências
    pip install -r requirements.txt

    # Iniciar o gateway
    python main.py
1

Cadastrar um usuário

Crie um usuário de teste. Anote o user_id retornado — ele será necessário nos passos seguintes.

bash 
python auth/sign_up/sign_up.py \
    --name "Test User" \
    --email "test.user@example.com" \
    --password "Test@1234" \
    --account-type "resident" \
    --rfid "RFID-1001" \
    --phone "+5511999999999"
account_typePerfil
residentCliente final / morador
comercialGestor do estabelecimento
adminAdministrador da plataforma
goodweAdministrador Master GoodWe
⚠️ Campos opcionais: phone e rfid. Se houver erro de validação, inclua ambos.
2

Fazer login

Autentique o usuário criado. O sistema retorna um token de sessão.

bash 
python auth/login.py \
    --email "test.user@example.com" \
    --password "Test@1234"
3

Consultar dados do usuário

Recupera nome, e-mail, tipo de conta e RFID pelo ID.

bash 
python auth/get_data_user.py --user-id user-0001
4

Cadastrar um veículo

Associe um veículo ao usuário criado no passo 1.

bash 
python modules/vehicles/register_vehicle.py \
    --user-id user-0001 \
    --plate ABC1D23 \
    --model "Leaf 40kWh" \
    --brand Nissan \
    --battery_capacity_kwh 40
5

Atualizar dados do veículo

Edite campos como apelido, placa ou capacidade da bateria.

bash 
python modules/vehicles/update_vehicle.py \
    --vehicle-id vehicle-0001 \
    --alias "Carro do João"
6

Listar veículos

Consulte todos os veículos de um usuário ou busque um específico.

bash 
# Todos os veículos do usuário
    python modules/vehicles/vehicle_meneger.py --list --user user-0001

    # Veículo específico
    python modules/vehicles/vehicle_meneger.py --vehicle-id vehicle-0001
7

Rodar a demonstração completa (Sprint 2)

Executa automaticamente 3 sessões simultâneas, a simulação DLM e o relatório final. Ideal para avaliação.

bash 
python demo.py

    # O script exibirá automaticamente:
    # → 3 sessões ativas (usuários 1000, 1001, 1002)
    # → Logs OCPP simulados
    # → Redistribuição DLM de sobrecarga
    # → Relatório: energia, faturamento, CO₂ evitado
