O sistema atende quatro perfis com níveis distintos de acesso e objetivos diferentes.

🏢

Administrador Master

Perfil GoodWe com acesso total. Monitora frota completa, visualiza toda a telemetria, gerencia royalties e tem acesso exclusivo ao módulo de IA.

📊

🏠

Cliente Residencial

Inicia recargas, consulta custos estimados e acompanha histórico com base no seu próprio carregador instalado.

Fundamentos

Os 4 Pilares da Solução

Cada pilar resolve um problema real da gestão de recarga em ambientes com múltiplos usuários e fontes de energia.

1
DLM

Controle de Demanda

Quando o consumo total se aproxima do limite contratado, o sistema reduz automaticamente a potência dos carregadores — cortando primeiro os usuários de menor prioridade para proteger a infraestrutura.

2
Interoperabilidade

Protocolos Abertos

O gateway traduz dados dos dispositivos físicos (MODBUS / SEMS Plus) para o formato OCPP simulado, garantindo compatibilidade com qualquer sistema de faturamento ou gestão de terceiros.

3
Financeiro

Tarifação Dinâmica

O custo do kWh varia conforme:

  • Horário (pico / fora de pico)
  • Tipo de usuário (Premium, Cortesia Solar…)
  • Disponibilidade de energia solar no momento
4
IA · Exclusivo Admin Master

Inteligência Aplicada

Demonstra o funcionamento base do modelo preditivo planejado para o sistema real. Composto por:

  • Modelo preditivo — antecipa picos de consumo nas próximas horas
  • Síndico Virtual — gera relatórios gerenciais em linguagem natural a partir da telemetria

Algoritmo Central

Controle de Demanda (DLM) e Energia

O DLM é o coração do sistema. Ele decide em tempo real quanta potência cada veículo recebe, com base no consumo total do prédio e na prioridade de cada usuário.

📡

Coleta (pull)

Gateway consulta SEMS Plus e medidores a cada ciclo. Captura kWh, tempo e estado do conector.

🔄

Cálculo DLM

Soma demanda total das sessões ativas. Compara com o limite contratado do prédio.

⚖️

Redistribuição

Se > 95% do limite, corta potência por ordem de prioridade de plano do usuário.

💰

Tarifação

Custo calculado após sessão: kWh × tarifa(horário, plano)

Dados & Auditoria

Sessões de Recarga e Logs OCPP

As sessões registram todo o fluxo de recarga — base para faturamento, gestão operacional e monitoramento de demanda. Os logs garantem rastreabilidade de cada evento crítico.

Formato de uma sessão

Cada recarga gera um objeto com os campos abaixo, armazenado para faturamento e relatórios:

json · sessão de recarga 
{
    "id":                    1,
    "usuario_id":           1000,
    "veiculo_id":           2000,
    "carregador_id":        2,
    "energia_consumida_kwh": 22.5,
    "custo_total":          11.25,
    "inicio":               "2026-06-12T18:30:00",
    "fim":                  "2026-06-12T20:00:00",
    "status":               "finalizada"
    }

Logs OCPP Simulado

Cada evento crítico gera um registro em JSON compatível com o padrão OCPP — pronto para integrar com qualquer sistema de faturamento:

OCPP-sim · evento ao vivo JSON 
{
    "id":           1,
    "carregador_id": 2,
    "tipo":          "BootNotification",
    "mensagem":      "Carregador conectado",
    "timestamp":     "2026-06-12 18:29"
    }
tipoQuando é gerado
BootNotificationAo inicializar o gateway ou reconectar um carregador
StartTransactionQuando um usuário inicia uma sessão de recarga
MeterValuesPeriodicamente durante a sessão — registra kWh acumulado
StopTransactionAo encerrar a sessão — contém resumo final de energia e custo
Execução

Como Rodar Localmente

Pré-requisito: Python 3.8 ou superior instalado.

bash · instalação 
# 1. Entre na pasta do projeto
    cd caminho/para/o/projeto

    # 2. Crie um ambiente virtual (recomendado)
    python -m venv .venv
    .venv\Scripts\activate      # Windows
    source .venv/bin/activate   # Linux / Mac

    # 3. Instale as dependências
    pip install -r requirements.txt

    # 4. Inicie o gateway
    python main.py

Demonstração automática (Sprint 2)

O demo.py executa os principais cenários sem nenhuma interação manual:

bash 
python demo.py

    # O que acontece automaticamente:
    # 1. 3 sessões simultâneas iniciadas (usuários 1000, 1001, 1002)
    # 2. Logs OCPP simulados exibidos (Boot / Start / MeterValues)
    # 3. Simulação DLM de sobrecarga e redistribuição de potência
    # 4. Relatório final: energia consumida, faturamento, CO₂ evitado
Fluxo de Uso

Tipos de Cadastro

O sistema possui três tipos de conta. O fluxo de cadastro varia conforme o perfil escolhido.

1

Cadastro de usuário

Dados comuns a todos os perfis. O campo account_type define o fluxo que será ativado a seguir.

account_typePerfil
cliente_residencialCliente com carregador próprio em casa
cliente_comercialCliente que recarrega em estabelecimentos
dono_estabelecimentoGestor de condomínio, empresa ou ponto de recarga
json · dados do usuário 
{
    "name":         "Test User",
    "email":        "test.user@example.com",
    "phone":        "11999999999",
    "password":     "Test@1234",
    "city":         "São Paulo",
    "account_type": "cliente_residencial"
    }
ℹ️ Se o tipo for dono_estabelecimento, o sistema solicitará os dados do estabelecimento após o cadastro do usuário.
json · dados do estabelecimento (dono_estabelecimento) 
{
    "name":                    "Condomínio Solar",
    "address":                 "Rua das Flores, 123",
    "cnpj":                    "45917263000106",
    "energia_solar":           "S",
    "demanda_contratada_kw":   44,
    "quantidade_carregador":   4
    }
2

Login

Autentique-se com e-mail e senha. O sistema retorna o token de sessão para as operações seguintes.

json · login 
{
    "email":    "maria.oliveira@email.com",
    "password": "123456"
    }
3

Cadastro de veículo

Caso não haja veículo cadastrado, o sistema solicitará o cadastro antes de iniciar a primeira recarga. O sistema exibe os modelos disponíveis — se o modelo não estiver na lista, o usuário pode cadastrá-lo manualmente:

json · cadastro de veículo 
{
    "usuario_id":  "id_do_usuario_logado",
    "modelo":      "BYD Dolphin",
    "fabricante":  "BYD",
    "placa":       "HJC2D33",
    "bateria_kwh": 44,
    "nivel_bateria": 40
    }
